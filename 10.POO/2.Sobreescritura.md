## üîπ ¬øQu√© es la sobrescritura?

La **sobrescritura** ocurre cuando una **clase hija redefine un m√©todo que ya existe en la clase padre**.

- El nombre del m√©todo debe ser el mismo.
    
- La versi√≥n de la subclase reemplaza la de la superclase.
    
- Permite cambiar el comportamiento **heredado**.


```python
class Animal:
    def sonido(self):
        return "Sonido gen√©rico"

class Perro(Animal):
    def sonido(self):  # Sobrescribe al m√©todo de Animal
        return "Guau!"

class Gato(Animal):
    def sonido(self):
        return "Miau!"

```

Aqu√≠, el m√©todo `sonido` existe en **Animal**, pero **Perro** y **Gato** lo **sobrescriben** con su propio comportamiento.

## üîπ ¬øQu√© pasa si no sobrescribo?

Si la clase hija no lo sobrescribe, simplemente **usa el m√©todo heredado**.

```python

    pass

print(Pajaro().sonido())  # Usa el de Animal ‚Üí "Sonido gen√©rico"
```


## üîπ Usando `super()` en la sobrescritura

A veces no queremos **reemplazar todo**, sino **ampliar lo que hace el padre**. Para eso usamos `super()`.

```python
class Animal:
    def sonido(self):
        return "Sonido gen√©rico"
```

```python
class Ave(Animal):
    def sonido(self):
        base = super().sonido()   # Llama al m√©todo de Animal
        return base + " (pero m√°s melodioso)"
        
        

print(Ave().sonido())  # "Sonido gen√©rico (pero m√°s melodioso)"
```


## üîπ Sobrescritura con `__init__`

No solo se sobrescriben m√©todos normales, tambi√©n el **constructor `__init__`**.

```python
class Persona:
    def __init__(self, nombre):
        self.nombre = nombre

class Estudiante(Persona):
    def __init__(self, nombre, carrera):
        super().__init__(nombre)   # Llamamos al __init__ de Persona
        self.carrera = carrera
```

```python
e = Estudiante("Ana", "Ingenier√≠a")
print(e.nombre)   # Ana
print(e.carrera)  # Ingenier√≠a
```

## üîπ Diferencia entre sobrecarga y sobrescritura

- **Sobrescritura (override):** Cambiar el comportamiento de un m√©todo heredado.
    
- **Sobrecarga (overload):** Tener varios m√©todos con el mismo nombre pero diferentes par√°metros (en Python no existe de forma nativa, se simula con valores por defecto o `*args, **kwargs`).
- 
üëâ En resumen:

- La **sobrescritura** permite que una clase hija cambie el comportamiento de un m√©todo padre.
    
- Con `super()` puedes **usar y extender** la l√≥gica del padre.
    
- Es la base del **polimorfismo**: diferentes clases pueden responder de forma distinta al mismo m√©todo.
