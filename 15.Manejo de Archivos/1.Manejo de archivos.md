
# Manejo de archivos

## üìÇ 1. Abrir un archivo

En Python se usa la funci√≥n **`open()`**:

- Este m√©todo nos ayuda con un archivo ya existente y si no existe se crea de manera autom√°tica
- Tambi√©n podemos pasar ciertos par√°metros, como para leer un archivo, escribir informaci√≥n a un archivo
- En este caso crearemos el archivo de "prueba" con su respectiva extensi√≥n, y el par√°metro "w" de write o "escribir"
- Como este c√≥digo puede lanzar una excepci√≥n lo com√∫n es que este dentro de un "try catch"
- encoding='utf8' nos ayuda a que el texto se muestre con acentos o caracteres especiales

```python
#En el caso de que el archivo ya exista este no creara uno nuevo
try:
    archivo = open('prueba.txt', 'w', encoding='utf8')
    archivo.write('Texto de prueba')
except Exception as e:
    print(e)
finally:
    archivo.close()
```


### colocar la ruta

En **Windows** las rutas de archivos pueden dar problemas porque el **backslash** (`\`) es un car√°cter especial en Python (sirve para secuencias como `\n`, `\t`, etc.). por eso se usa la doble barra invertida.

```python
archivo = open('C:\\Users\\ujdan\\Documents\\PYTHON\\prueba.txt', 'r')
```

Lo m√°s simple suele ser usar la opci√≥n **3 con `/`**, porque funciona en Windows, Linux y macOS sin cambiar nada.

```python
archivo = open('C:/Users/ujdan/Documents/PYTHON/prueba.txt', 'r')
```

>Pero si estamos dentro de la misma carpeta no es necesario especificar toda la ruta

### Par√°metros principales de `open`:

- `"r"` ‚Üí **read** (lectura, el archivo debe existir).
    
- `"w"` ‚Üí **write** (escritura, si no existe lo crea, si existe lo sobrescribe).
    
- `"a"` ‚Üí **append** (agregar al final, sin borrar lo anterior).
    
- `"x"` ‚Üí **exclusive creation** (crear un archivo nuevo, da error si ya existe).
    
- `"b"` ‚Üí modo binario (ejemplo: `"rb"`, `"wb"`) para im√°genes o archivos binarios.
    
- `"t"` ‚Üí modo texto (por defecto).
