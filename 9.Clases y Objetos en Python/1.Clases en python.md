# Clases en python

En Python, **una clase** es como un molde o plantilla que te permite crear objetos con ciertas propiedades (atributos) y comportamientos (m√©todos).

Piensa que una clase es como el plano de una casa:

- El **plano** (la clase) describe c√≥mo ser√° la casa.
    
- La **casa construida** (el objeto o instancia) es algo real que puedes usar.


## Partes importantes

- **`class`** ‚Üí Palabra clave para definir una clase.
    
- **`__init__`** ‚Üí M√©todo especial llamado _constructor_, se ejecuta autom√°ticamente al crear el objeto.
    
- **`self`** ‚Üí Es una referencia a la propia instancia; permite acceder a sus atributos y m√©todos.
    
- **Atributos** ‚Üí Datos que pertenecen al objeto.
    
- **M√©todos** ‚Üí Funciones que pertenecen al objeto.

## Ventajas de usar clases

- **Organizaci√≥n**: agrupa datos y funciones relacionadas.
    
- **Reutilizaci√≥n**: puedes crear muchas instancias sin reescribir c√≥digo.
    
- **Herencia**: puedes crear clases nuevas basadas en otras.
    
- **Encapsulaci√≥n**: controlas qu√© partes del c√≥digo pueden acceder a ciertos datos.


## Self

En Python, cuando defines m√©todos dentro de una clase, el primer par√°metro siempre es self, que representa la instancia actual del objeto.

Un par√°metro de un m√©todo solo ‚Äúvive‚Äù mientras ese m√©todo se est√° ejecutando.  
Ejemplo:

```python
class Persona:
    def metodo1(self, nombre):
        print(nombre)  # <- v√°lido aqu√≠
    def metodo2(self):
        print(nombre)  # <- ERROR, nombre ya no existe
```

üëâ Aqu√≠, `nombre` desaparece cuando termina `metodo1`.

### C√≥mo hacer que sean accesibles en cascada

Si quieres que un valor que llega como par√°metro en un m√©todo est√© disponible en otros m√©todos, tienes que **guardarlo como atributo del objeto** usando `self`.

```python
class Persona:
    def metodo1(self, nombre):
        self.nombre = nombre   # lo guardo en el objeto
    def metodo2(self):
        print(self.nombre)     # ya lo puedo usar aqu√≠

```


üëâ En resumen:

- Si un par√°metro **no lo guardas con `self`**, solo vive en el m√©todo que lo recibi√≥.
    
- Si lo guardas en `self`, cualquier m√©todo de la clase puede usarlo.


Ejemplo sencillo con **dos m√©todos que reciben par√°metros** y c√≥mo puedes hacer que ambos compartan la informaci√≥n a trav√©s de `self`.

```python
class Persona:
    def inicializar_persona(self, nombre, apellido):
        # Guardamos en el objeto (self)
        self.nombre = nombre
        self.apellido = apellido

    def agregar_edad(self, edad):
        # Tambi√©n guardamos en el objeto
        self.edad = edad

    def mostrar_persona(self):
        # Aqu√≠ ya podemos acceder a TODOS los atributos guardados con self
        print(f'''
        Persona:
        Nombre: {self.nombre}
        Apellido: {self.apellido}
        Edad: {self.edad}
        ''')
        

# Ejemplo de uso
persona1 = Persona()
persona1.inicializar_persona("Layla", "Gomez")  
persona1.agregar_edad(25)  
persona1.mostrar_persona()

```
